- confirm no remaining call site uses a COUNT for the suffix.
- confirm the accepted-peak carry-forward (aug_report.proposed_peaks =

DISPOSITION 2 (MAJOR, run B — F2 budget overrun). An augmented
fit_candidate has no internal wall clock (~10-12 s worst-case at
FIT_CANDIDATE_MAX_NFEV), and the proposal pass could START one with ~1 s
of sweep budget left, overrunning TOTAL_ANALYSIS_TIMEOUT_SEC (240) and
hence the gunicorn --timeout (300). Also the stability deadline was
computed from a STALE budget_remaining snapshot taken before that fit.
FIXED in _attempt_proposal:
- new constant PROPOSAL_MIN_FIT_BUDGET_SEC = 15.0; a guard at the top of
  _attempt_proposal fast-rejects ('insufficient_budget') when
  budget_remaining < PROPOSAL_MIN_FIT_BUDGET_SEC, BEFORE the augmented fit.
- attempt_start = time.perf_counter() at entry; the stability deadline now
  uses remaining = budget_remaining - (now - attempt_start) (dynamic, not
  the stale snapshot), and fast-rejects if remaining <= 0.
- the caller passes budget_remaining = min(pass budget, sweep budget left)
  as before (pass_budget = min(PROPOSAL_CANDIDATE_TIMEOUT_SEC 60,
  TOTAL - offset)), so the guard bounds worst-case wall.

- confirm the dynamic `remaining` cannot go negative-then-positive or
