CONTEXT: Find Peaks (the automated peak-fitting feature, distinct from
manual Run Fit / /api/fit, which this change must not touch) resolves a
region to a "grammar" of candidate models. Each curated grammar module
(autofit/regions/c1s.py, n1s.py, cl2p.py, b1s.py, u4f.py) defines
per-component slots with a FIXED, literature-anchored `be_window` that
hard-bounds where lmfit can place that component's center parameter —
completely independent of how wide the user's ROI is. That's the right
default (C 1s has 6 chemically-distinct sub-windows, e.g. graphitic
284.0-284.8, aliphatic 284.6-285.4, ... shake-up 290.0-292.0 — each
anchors a specific chemical state), but it means an unusually-shifted
extreme component just outside the outermost window is unreachable no
matter how wide the user sets their ROI.

THE FIX (opt-in, default OFF — must not change today's behavior unless
explicitly enabled): a new `fit_full_window: bool = False` option.
IMPORTANT DESIGN NUANCE (resolved via an explicit design conversation
with the repo owner before implementation — verify the code actually
does this, don't take it on faith): the widening branches on how EACH
SLOT was solved, not on whether the region has grammar:
- a slot with `region == "unassigned"` (a detection/structural-fallback
  slot — e.g. Fe 2p, or an out-of-grammar "preseed" slot added
  alongside curated slots in the same model) has no cited per-component
  window to preserve — widens FULLY to the ROI on both sides.
- any other (curated) slot widens ONLY the OUTER ENVELOPE: among all
  curated PRIMARY (non-linked) slots in one candidate model, the
  lowest-be_window[0] slot's lower bound extends to the ROI's low edge,
  and the highest-be_window[1] slot's upper bound extends to the ROI's
  high edge — every OTHER (interior) slot, and the untouched side of an
  outer slot, must keep its original literature window byte-for-byte.
  This is the crux of the whole design: it must be IMPOSSIBLE for this
  option to let one chemical-state component (e.g. C-O) wander into a
  neighboring one's territory (e.g. graphitic).
- LINKED slots (spin-orbit doublet partners, satellites — anything with
  `linked_to` set) must NEVER be touched by this option at all: their
  position comes from a `parent_center + offset` expression, and the
  offset range is a cited physical splitting, unrelated to ROI cropping.
- The optimizer's STARTING GUESS (the initial `value=` passed to
  lmfit) and the amplitude-estimate window must ALSO stay anchored to
  the slot's own original `be_window` even when the hard bound widens —
  only the constraint relaxes, never where the search starts.
