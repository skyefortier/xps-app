**Findings**

- **BLOCKER** [autofit/methods/sparse_map.py](/Users/skyefortier/xps-app/autofit/methods/sparse_map.py:72): `_build_dictionary` deduplicates slots by `role` with `setdefault`, so it does not build the documented union of grammar slot variants. Failure scenario: N 1s has `main_n1s` as both PV and ASYM_GL; the first PV slot wins, so `asymmetric_slots_not_expressible` misses the ASYM_GL variant. C 1s variants with wider lab FWHM ranges under the same role can also be silently excluded. Fix: build from all distinct `(role, region, phase_id, be_window, fwhm_range, line_shape)` slot variants, or explicitly union windows/ranges per role while preserving an `any asymmetric` flag.

- **BLOCKER** [autofit/methods/sparse_map.py](/Users/skyefortier/xps-app/autofit/methods/sparse_map.py:200): clustering and BIC use the LASSO support even after debiased NNLS may set some support amplitudes to zero. Failure scenario: a zero-amplitude atom remains in `idx`, counts in `k`, and can bridge clusters because `_cluster_support` uses center gaps to `cl[-1]` at [line 135](/Users/skyefortier/xps-app/autofit/methods/sparse_map.py:135). That can merge two real peaks or chain-merge through a broad/zero atom. Fix: after NNLS, define active support as `amps > eps`, recompute `k`, RSS, and clusters on active atoms only.

- **MAJOR** [autofit/methods/sparse_map.py](/Users/skyefortier/xps-app/autofit/methods/sparse_map.py:114): the coordinate update is mathematically correct for unit-norm non-negative LASSO, and the residual update is algebraically exact apart from floating roundoff. But convergence is only `max |Δa_j| < tol`; there is no KKT residual check and no flag if `cd_max_iter` is exhausted. Failure scenario: highly collinear Gaussian atoms stop with tiny coordinate movement while KKT violations still affect support, BIC, and clustering. Fix: check `max` KKT violation, recompute residual periodically or at exit, and record/raise non-convergence.

- **MAJOR** [autofit/methods/sparse_map.py](/Users/skyefortier/xps-app/autofit/methods/sparse_map.py:275): BIC on debiased NNLS support with `k = selected atoms` is defensible only as a fixed-dictionary heuristic, not calibrated evidence. It breaks when one physical peak is represented by many atoms, when atoms are collinear so effective dof is below atom count, and after lambda-path selection on the same data. Fix: label it as “heuristic BIC on active dictionary atoms,” surface post-selection optimism, and consider reporting both `n_atoms_active` and `n_clusters`.

- **MINOR** [autofit/methods/sparse_map.py](/Users/skyefortier/xps-app/autofit/methods/sparse_map.py:192): the comment says `dense→sparse`, but the path decreases from `0.999 * lambda_max` to smaller lambda, so it is sparse-to-dense. `lambda_max = max(A.T @ y)` is correct for the non-negative empty-support threshold. Warm starts do not break determinism, but with collinear atoms and finite convergence they can lock in arbitrary support choices. Fix the comment and add KKT-based convergence.

- **MAJOR** [tests/autofit/test_sparse_map.py](/Users/skyefortier/xps-app/tests/autofit/test_sparse_map.py:57): tests are mostly end-to-end and can pass with a subtly wrong solver. A lambda scale error might still recover two peaks after BIC; an ordinary soft-threshold without the non-negative clamp is not pinned on negative correlations. Fix: add a private analytic test for `_nn_lasso_cd`, e.g. `A = I`, `y = [2.0, -2.0, 0.3]`, `lambda = 0.5`, expected `[1.5, 0.0, 0.0]`, plus `lambda >= max(A.T@y)` yields empty support and `lambda just below` activates the max-correlation atom.

The gap is the slot-dedup bug above, which can hide asymmetric inexpressibility despite the payload claiming to flag it.
